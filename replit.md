# QuickQuote - SaaS Quote Calculator Builder

## Overview
QuickQuote is a SaaS application designed to empower trades businesses (e.g., plumbing, concrete, cleaning) by providing tools to create instant quote calculators. These calculators can be embedded on their websites to generate real-time estimates for customers and efficiently collect leads. The project aims to offer a responsive, mobile-first experience with a premium, neutral SaaS aesthetic.

## System Architecture
The application features a decoupled architecture with a React + TypeScript frontend utilizing Wouter for routing, Tailwind CSS for styling, and Shadcn UI components. The backend is built with Express.js, providing API routes for various functionalities. Data persistence is managed by a PostgreSQL database, hosted via Neon on Replit. AI capabilities, specifically for pricing generation, are integrated using OpenAI (gpt-5-mini) through Replit AI Integrations. User authentication for editing and accessing leads is token-based, with tokens expiring after 7 days, avoiding the need for traditional user accounts.

**UI/UX and Design Decisions:**
- **Mobile-first Design:** All layouts are primarily designed for mobile screens (e.g., 480px max-width wizard card) and are fully responsive for desktop.
- **Card-based Layouts:** Utilizes soft shadows and elevated cards for a clean, modern aesthetic.
- **Theme Architecture:**
    - **PlatformTheme (Builder UI):** A generic premium SaaS aesthetic for the wizard builder, edit pages, and leads dashboard, defined in `client/src/theme/platformTheme.ts`. It uses a sage accent (`#2D6A4F`) sparingly, neutral whites/grays for surfaces, and soft shadows. This theme is strictly isolated and never affected by client widget theme changes.
    - **WidgetTheme (Customer Calculator):** This theme is per-client, allowing each calculator to have a custom appearance. It's defined in `client/src/theme/widgetTheme.ts` with a default sky blue accent (`#0284C7`) and is applied only within the `.widget-scope` class for CSS isolation.
- **Typography:** Uses the Inter font stack with specific color values for headings, body text, and muted text.
- **Interactivity:** Minimum 44px touch targets on mobile, full-width inputs, CSS keyframe animations for transitions, and utility classes (`hover-elevate`, `active-elevate-2`) for interactive elements. Lucide-react icons are used instead of emojis.
- **Premium Inputs:** Custom `.premium-input` class with sage focus states (border + ring).
- **Wizard Flow:** A 6-step wizard guides users through calculator creation: Business & Trade Setup, Design Your Calculator, Pricing Logic, Capture Leads, Validate Your Pricing, and Publish & Share. State is persisted in `localStorage`.
- **Lead Form (Step 3 — `LeadFormStep.tsx`):** Premium lead capture builder with mode selector (optional/gated/call-only), 9 toggleable fields (name, phone, email, address, city, postal/ZIP, date/time, job notes, file upload), consent/compliance section with editable text, CTA button customization, collapsible delivery settings (primary/secondary email), collapsible anti-spam (honeypot enabled by default). Includes live preview updating in real-time. Data stored as versioned JSON in `calculator_settings.lead_form`. Validation gate blocks continue without delivery email or fields in gated mode. Pro features (file upload, SMS consent, webhook, reCAPTCHA, CRM) are shown but disabled.
- **Publish & Share (Step 5 — `PublishStep.tsx`):** Premium publish step with scalable deployment architecture. Features 4 deployment option tabs: (1) **Instant Hosted Page** — auto-generated subdomain `{slug}.estimate.ai` with SSL, copy/open buttons; (2) **Embed on Website** — Script/Iframe/Button snippets with installation guides (WordPress/Shopify/Custom HTML); (3) **Custom Domain (Pro)** — locked UI for entering custom domain, DNS CNAME verification workflow with auto-check every 30s, SSL auto-provisioning, status progression (pending_dns → dns_verified → ssl_provisioning → active); (4) **Done-For-You Install (Pro)** — locked UI for professional installation service. Includes status badge (Draft/Ready/Live), readiness checklist, subdomain banner, health indicators (hosted page, pricing, lead capture, anti-spam, SSL, custom domain), republish banner, and "Create Another Calculator" reset. Slugs are auto-generated from business name via shared `slugify()` utility (`shared/slugUtils.ts`). Publish data stored in `calculator_settings.publish` with version/status/slug/subdomain/published_at/embed_id/last_modified/custom_domain/custom_domain_status/ssl_status/last_dns_check/hosting_domain. Backend endpoints: `GET /api/calculators/check-slug`, `GET /api/calculators/slugify`, `POST /api/domains/check-dns`, `POST /api/domains/issue-ssl`, `GET /api/domains/status`.
- **Test Gate (Step 4 — `TestGateStep.tsx`):** Premium validation system with 3 expandable scenario cards (Small/Typical/Large Job), real-time estimate calculation using `calculateEstimate.ts`, "What would YOU charge?" comparison with auto-deviation % and color-coded indicators (green ≤10%, yellow 10-20%, red >20%), accuracy score meter (0-100), smart suggestions for large deviations, Advanced Adjustment Mode for baseFee/rate/minCharge/multiplier, confirmation checkbox, and a strict publish gate (3 filled scenarios + accuracy ≥60 + 2+ within ±20% + checkbox). Call-for-quote/price-range types bypass scenario requirements. Test state persists via `test_history` in `calculator_settings`.
- **Pricing Architecture:**
    - Employs 10 strict "formula families" for all pricing calculations, defined and validated in `shared/pricingConfig.ts` with Zod schemas. No custom math is allowed.
    - A runtime calculator engine, `shared/calculateEstimate.ts`, handles all estimate calculations.
    - AI pricing generation (`/api/ai/generate-pricing`, `/api/ai/generate-pricing-draft`) is constrained to these formula families and validated.
    - A "Universal Pricing Questions" system (two-stage intake) in `CustomTradeQuestionnaire.tsx` and `PricingIntakeStage2.tsx` allows for structured input, which is mapped to `PricingConfigV1` by `shared/pricingIntakeMapper.ts`. AI is used as a fallback when the charge method is uncertain or mapping fails.
    - An AI Agent (`server/aiPricingAgent.ts`) is designed as a constrained composer with strict prompts, few-shot examples, derivation-first logic, and multi-layer validation to ensure valid pricing configurations.
- **SaaS Dashboard (`/Dashboard?token=...` — `client/src/pages/dashboard.tsx`):** Clean 5-section dashboard with sidebar navigation. (1) **Overview** — status badge (Draft/Live), hosted link with copy/open, custom domain status, stat cards (leads this week, conversion rate, total views, avg quote), quick-nav buttons. (2) **Pricing** — Quote Confidence level display, Edit Pricing and Re-run Validation buttons. (3) **Leads** — table view (Name/Phone/Email/Quote/Date), search, CSV export, delete lead. (4) **Analytics** — stat cards (views, leads, conversion rate, avg quote), weekly trend bar chart. (5) **Settings** — notification email, SMS toggle (Pro), custom domain info, republish, delete calculator. Accessed via edit_token, link available from PublishStep "Go to Dashboard" button.
- **Automation:** Auto-tracking of analytics events (views, leads) wired into existing routes. Auto-republish on pricing changes when `auto_republish` enabled in `deployment_status`. Background analytics aggregation via `analytics_events` table with weekly trend queries.
- **Database Tables:** `calculators` (main config, single source of truth via `calculator_settings` JSONB), `leads` (captured leads), `analytics_events` (view/lead/quote events with timestamps), `deployment_status` (per-calculator deployment state, auto-republish flag).
- **Data Handling:** All API routes use Zod validation schemas. Frontend data fetching uses `@tanstack/react-query`. Token expiry is enforced on all token-gated routes.
- **Analytics & Automation Layer (`server/jobs/`):**
    - **Event Tracking:** Enhanced `POST /api/dashboard/track` and `POST /api/calculators/track-view` endpoints track page views, quote generated, lead submitted, confidence tier, and device type (mobile/tablet/desktop) with metadata stored in `analytics_events`.
    - **Daily Aggregation Job (`server/jobs/aggregation.ts`):** Runs at 02:00 UTC daily via `node-cron`. Aggregates per-calculator stats (total_views, total_quotes, total_leads, conversion_rate, avg_quote_value, best_day) into `calculator_analytics_summary` table.
    - **Weekly Email Report (`server/jobs/weeklyReport.ts`):** Runs at 13:00 UTC every Monday (~8AM EST) via `node-cron`. Sends HTML email with views, leads, conversion rate, avg quote, best day, and CTA "Edit Pricing to Improve Conversions". Uses Nodemailer with SMTP config (SMTP_HOST, SMTP_PORT, SMTP_USER, SMTP_PASS, SMTP_FROM). SMS summary is Pro-only stub.
    - **Job Scheduler (`server/jobs/scheduler.ts`):** Central scheduler with retry logic (3 retries, exponential backoff), failure logging to `job_logs` table, and startup initialization in `server/index.ts`.
    - **Notification Worker (`server/jobs/notificationWorker.ts`):** Runs every minute via cron. Processes queued instant business alerts (email via Nodemailer, webhook stub). Rate limited to 30 notifications/hour per calculator. Retry logic with 3 attempts and exponential backoff. Supports email and webhook notification types.
    - **Follow-Up Worker (`server/jobs/followupWorker.ts`):** Runs every minute via cron. Processes scheduled follow-up drip sequences (thank_you at 2min, reminder at 24hr, last_call at 3 days). Re-checks calculator settings at send-time to respect disabled state. Stops sequence if lead status is no longer "New". Template variable substitution with {{name}}, {{quote_amount}}, {{business_name}}, {{phone}}, {{booking_link}}, {{service_area}}.
    - **Database Tables:** `calculator_analytics_summary` (aggregated daily stats), `job_logs` (job execution history with status/errors), `notification_queue` (instant notification jobs with status/retry tracking), `followup_jobs` (scheduled follow-up drip jobs with channel/template/timing).
- **Lead Status Management:** Leads have a status field (new/contacted/won/lost). Changing status from "new" cancels pending follow-up jobs. Status is editable via dropdown badges in the Dashboard Leads table.
- **Dashboard Follow-Up Tab:** 6th sidebar section with: Autopilot enable/disable toggle, personalization fields (business_name, phone, booking_link, service_area), 3-message email sequence editor with expandable template cards, test send per template, and recent activity log showing notification/followup history.
- **API Routes for Follow-Up:** `PATCH /api/dashboard/leads/:id/status` (update lead status), `GET /api/dashboard/followup` (get settings), `PUT /api/dashboard/followup` (save settings), `POST /api/dashboard/followup/test` (send test email), `GET /api/dashboard/notification-logs` (activity logs).

## External Dependencies
- **PostgreSQL:** Used as the primary database, backed by Neon via Replit.
- **OpenAI:** Integrated via Replit AI Integrations, specifically using `gpt-5-mini` for AI-powered pricing generation and drafting.
- **Wouter:** A small routing library used in the React frontend.
- **Tailwind CSS:** A utility-first CSS framework for styling the application.
- **Shadcn UI:** A collection of reusable components built with Tailwind CSS.
- **Lucide-react:** An icon library used for UI elements.
- **@tanstack/react-query:** Used for data fetching, caching, and state management in the frontend.
