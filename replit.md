# QuickQuote - SaaS Quote Calculator Builder

## Overview
QuickQuote is a SaaS application that enables trades businesses (e.g., plumbing, concrete, cleaning) to build and embed instant quote calculators on their websites. It facilitates real-time estimate generation for customers and efficient lead collection. The project aims to deliver a responsive, mobile-first experience with a premium, neutral SaaS aesthetic, enhancing lead generation and customer engagement for these businesses.

## System Architecture
The application features a decoupled architecture comprising a React + TypeScript frontend with Wouter for routing, Tailwind CSS for styling, and Shadcn UI components. The backend is built with Express.js, providing API routes. Data is persisted in a PostgreSQL database. AI capabilities for pricing generation are integrated using OpenAI (gpt-5-mini) via Replit AI Integrations. User authentication for editing and accessing leads is token-based.

**UI/UX and Design Decisions:**
- **Mobile-first Design:** Layouts are primarily designed for mobile and are fully responsive.
- **Card-based Layouts:** Utilizes soft shadows and elevated cards for a modern aesthetic.
- **Theme Architecture:**
    - **PlatformTheme (Builder UI):** A generic premium SaaS aesthetic for the builder, edit pages, and leads dashboard, using a sage accent and neutral whites/grays.
    - **WidgetTheme (Customer Calculator):** Per-client customizable theme with a default sky blue accent, isolated to `.widget-scope`.
- **Typography:** Uses the Inter font stack with specific color values.
- **Interactivity:** Minimum 44px touch targets on mobile, full-width inputs, CSS keyframe animations, and `Lucide-react` icons. Custom `.premium-input` class with sage focus states.
- **Wizard Flow:** A 6-step wizard (Business & Trade Setup, Design Your Calculator, Pricing Logic, Capture Leads, Validate Your Pricing, Publish & Share) guides calculator creation, with state persisted in `localStorage`.
- **Lead Form:** A premium lead capture builder with mode selection (optional/gated/call-only), 9 toggleable fields, consent/compliance section, CTA button customization, delivery settings, and anti-spam (honeypot). Pro features are displayed but disabled.
- **Publish & Share:** Offers 4 deployment options: Instant Hosted Page (`{slug}.estimate.ai`), Embed on Website (Script/Iframe/Button snippets), Custom Domain (Pro, with DNS verification and SSL provisioning), and Done-For-You Install (Pro). Includes status badges, readiness checklist, and health indicators.
- **Test Gate:** A premium validation system with 3 scenario cards (Small/Typical/Large Job), real-time estimate calculation, "What would YOU charge?" comparison, accuracy score meter, smart suggestions, and Advanced Adjustment Mode. Requires scenarios to be filled and accuracy thresholds met for publishing.
- **Pricing Architecture:** Employs 10 strict "formula families" defined and validated with Zod schemas. A runtime calculator engine handles all estimate calculations. AI pricing generation is constrained to these formula families. A "Universal Pricing Questions" system (two-stage intake) maps inputs to pricing configurations, with AI as a fallback. An AI Agent ensures valid pricing configurations through constrained composition, strict prompts, and multi-layer validation.
- **SaaS Dashboard:** A 5-section dashboard (Overview, Pricing, Leads, Analytics, Settings) accessible via `edit_token`, displaying calculator status, analytics, lead management, and configuration options. A Bookings tab is the 7th nav item for managing appointments.
- **Automation:** Auto-tracking of analytics events (views, leads), background analytics aggregation via a daily job, weekly email reports, and a central job scheduler with retry logic.
- **Notification & Follow-Up Workers:** Instant business alerts (email, webhook) and scheduled follow-up drip sequences (thank_you, reminder, last_call) are processed by dedicated workers. Lead status changes can cancel pending follow-ups.
- **Database Tables:** Core tables include `calculators` (main config), `leads`, `analytics_events`, `deployment_status`, `calculator_analytics_summary`, `job_logs`, `notification_queue`, `followup_jobs`, `bookings`, `ai_conversations` (AI chat transcripts by agent type), and `support_tickets` (platform support escalations).
- **AI Employee System (Phase 5):** 3 isolated AI agent types with server-enforced permissions. `demo_ai_employee` — sandbox demo using hardcoded trade presets, zero real DB access. `platform_support_ai` — authenticated dashboard users only, can create/escalate support tickets to ADMIN_EMAIL. `client_ai_employee` — embedded on public quote pages, accesses only that calculator's settings, uses OpenAI function calling for `generate_estimate`, `show_available_slots`, `create_booking`, `submit_lead`. Agent type is NEVER trusted from client — derived from endpoint server-side. 14-day free trial enforced server-side (403 `trial_expired` after expiry). Chat endpoints: `POST /api/ai/demo-chat`, `POST /api/ai/support-chat`, `POST /api/ai/client-chat`, `POST /api/ai/create-ticket`. Engine: `server/aiChatEngine.ts` with `ALLOWED_TOOLS` per agent type + `enforceToolPermissions()` guard. Floating `AIChatBubble` renders on public calculator page only when `ai_employee.enabled && status in (trial, active) && !isEmbed`. Standalone website embed: `client/public/embed-chat.js` (vanilla JS iframe injection). Training configured in wizard Publish step (structured form — no freeform prompt editing): business summary, services, service area, working hours, emergency service toggle, escalation contacts, tone selector.
- **Data Handling:** All API routes use Zod validation. Frontend data fetching uses `@tanstack/react-query`. Token expiry is enforced.
- **Estimate + Booking Hybrid Engine:** Supports `estimate_only`, `estimate_plus_booking`, or `booking_only` calculator types. Booking settings (deposit, slot duration, availability, Stripe Connect) are configured within the wizard. A "Book Now" button appears after estimates, leading to an inline booking panel with calendar/time slot selection. Requires Stripe Connect Express for deposit processing. Double-booking prevention is implemented. A "Bookings" tab in the Dashboard manages appointments with status updates. Confirmation emails are sent to customers and businesses.
- **Templates + Sliders System (Phase 2):** 6 high-converting UI templates for the customer-facing calculator widget, selectable in the Design Studio (wizard step 2). Templates: `classic_single` (default, preserves original UI), `classic_two_column` (sticky summary sidebar), `multi_step_progressive` (step-by-step with progress bar), `package_selector` (tiered package cards with "Most Popular" badge), `range_only_leadgate` (range display encouraging contact), `estimate_then_book` (booking CTA emphasis). Trade-based auto-selection recommends templates when a trade is chosen (e.g., cleaning → two-column, photography → package cards, booking enabled → estimate-then-book). Slider inputs replace number steppers where applicable (sq_ft, bedrooms, hours, etc.) with premium feel, floating value bubbles, and smooth transitions. All template settings stored in `ui_template` within `calculator_settings` JSON. Key files: `shared/templateLibrary.ts` (registry + trade mappings), `shared/sliderMappings.ts` (field-to-slider configs), `client/src/components/calculator/SliderField.tsx` (reusable slider component).
- **Conversion Blocks (Phase 3):** Optional images, testimonials, and trust badge blocks stored in `calculator_settings.conversion_blocks`. Each block has `enabled`, `placement` (top/under_title/near_cta/under_total/bottom), and `layout` (grid/carousel for images, cards/carousel for testimonials). Images: base64 data URLs (max 6, 5MB each, PNG/JPG/WebP), optional captions, reorderable. Testimonials: name, location, 1-5 star rating, text (20-240 chars), max 6, reorderable. Trust badges: 4 toggleable badges (insured/licensed/bonded/satisfaction) with optional microcopy (max 80 chars). All configured in Design Studio Conversion tab via collapsible sections (Photos, Reviews, Trust Badges). Widget renders blocks at configurable placements across all 6 template layouts using `renderPlacementSlot()` helper. Schema: `conversionBlocksSchema` in `shared/schema.ts`.
- **Conversion Automation Engine (Phase 4):** Coupon system (`calculator_settings.promotions`) with percentage/fixed discount codes, expiry dates, usage limits, and per-lead redemption tracking. Public coupon validation API at `POST /api/calculators/:slug/coupons/validate`. Widget shows collapsible promo code input, applies discount in real-time with breakdown line, passes coupon to lead submission. Quote expiration (`calculator_settings.quote_rules`) with configurable `valid_days` and optional countdown badge — purely client-side. Dashboard analytics extended with Bookings, Payments, Coupon Uses stats and a conversion funnel (Views→Leads→Bookings→Payments). Design Studio has a new "Automation" tab (5th tab) managing Promo Codes, Quote Expiration, and Follow-Up Reminders settings. Follow-up last_call emails can include a discount coupon code (`reminder_2_include_discount` + `reminder_2_coupon_id`). Follow-up schedule extended with `reminders` sub-object for per-reminder delay/enable config.
- **Multi-Channel AI Messaging (Phase 6):** SMS + WhatsApp via Twilio. Same `client_ai_employee` brain + tool guardrails as Phase 5. `calculator_settings.ai_employee` extended with `channels` (web_chat/sms/whatsapp toggles), `twilio` (from_number, whatsapp_number), and `consent` (sms_required, consent_text, store_consent). New `sms_messages` DB table stores all inbound/outbound threads. `leads` table extended with `consent_timestamp`, `consent_text_version`, `ai_paused`. Inbound webhook at `POST /api/twilio/inbound` routes by lead phone number, checks AI pause + rate limits (3/lead/day, 50/business/day), replies via TwiML. Followup worker extended to actually send SMS via Twilio (replaces Pro placeholder). Dashboard gains "Messages" tab with thread view + "Take Over" button, and Settings > AI Employee card with channel toggles + SMS status. Calculator widget shows SMS consent checkbox when `ai_employee.channels.sms = true`. Key files: `server/twilioClient.ts` (Twilio client, rate limiting, phone matching), `server/jobs/followupWorker.ts` (SMS sending). Requires env vars: `TWILIO_ACCOUNT_SID`, `TWILIO_AUTH_TOKEN`, `TWILIO_FROM_NUMBER`, `TWILIO_WHATSAPP_NUMBER` (optional).

## External Dependencies
- **PostgreSQL:** Primary database, hosted via Neon.
- **OpenAI:** Used for AI-powered pricing generation (`gpt-5-mini`) via Replit AI Integrations.
- **Stripe:** For payment processing via Stripe Connect Express, enabling businesses to collect deposits.
- **Wouter:** Frontend routing library.
- **Tailwind CSS:** Utility-first CSS framework.
- **Shadcn UI:** Reusable UI components.
- **Lucide-react:** Icon library.
- **@tanstack/react-query:** For frontend data fetching and state management.
